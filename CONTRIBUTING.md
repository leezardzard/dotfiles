# Contributing

Issues and pull requests are welcome.

## How to contribute

1. **Fork** the repo and clone your fork.
2. **Branch** — create a branch for your change (e.g. `feature/add-xyz`, `fix/bootstrap-xyz`).
3. **Change** — keep the scope small and focused; describe what you changed in the PR.
4. **Open a PR** against this repo’s default branch.

## Code and style

- Prefer shell-agnostic patterns where it makes sense; zsh-specific scripts stay in `.zsh` files.
- No formal style guide for now; match the existing style in the file you edit.

## Reporting issues

When opening an issue, please include:

- macOS version
- What you ran (e.g. `./scripts/bootstrap`, `./scripts/bootstrap-zshrc.zsh`)
- Relevant error output (paste the message or a short log)

That helps debug quickly.
